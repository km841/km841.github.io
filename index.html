Unreal Engine 5의 기능들을 최대한 활용하기 위한 토이 프로젝트


Lyra는 Epic Games에서 제공하는 공식 Document로 멀티플레이어 게임 제작의 기본 구조에 대해 익힐 수 있도록 설계되었습니다.


1. Experience - 9/27 ~ 10/2
- Lyra에는 섬멸전, 거점전, 데스매치 등 많은 모드가 존재하고 데디서버를 사용하는 Lyra 특성 상 모드 변경 시 부하를 최소화하도록 설계되었습니다.
- 그 기능 중 하나로 하나의 레벨 내에서 Experience라는 개념을 도입하여 끊김 없는 전환을 구현하였습니다.
- 이 기능을 구현하기 위해 Play 버튼을 눌렀을 때 캐릭터가 기존 언리얼 엔진 초기 세팅대로 Spawn되지 않고, 제가 원하는 시점에 Spawn되도록 구현하였습니다.
- 이 기회를 통해 엔진 내부 동작을 엿볼 수 있었습니다.


2. Pawn Extension - 10/2 ~ 10/19
- Pawn Extension은 확장 가능하고 모듈화된 게임플레이 구조를 구현하기 위해 설계되었습니다.
- 일전의 DirectX11 개발 경험을 통해 하나의 클래스가 비대해지는 경우, 특히 Player에 해당하는 코드는 많은 기능이 구현되어야 하므로 쉽게 복잡해집니다.
- Lyra에서는 이런 부분들을 모듈화함으로써 과도하게 복잡해지지 않도록 하였고, 해당 클래스가 Pawn Extension입니다.
- Pawn Extension은 Component 디자인 패턴으로 구현되며, 이후 등장할 AbilitySystem과 조합되도록 구현했습니다.


3. Game Features - 10/19 ~ 10/31
- Game Features 기능 또한 모듈화를 위한 기능으로써 Weapon인 Pistol을 GameFeature로 만들어서, Player가 Spawn될 때 손에 쥐어주도록 했습니다.
- 다만 이 시점의 Weapon은 실제 격발되지 않는 껍데기에 불과했습니다.


4. Weapon Details - 10/31 ~ 11/29
- Weapon 구조를 파악하며 이해하기 위해 노력했습니다.
- Weapon Instance -> Ranged Weapon Instance의 상속 구조로 되어 있으며 해당 클래스는 Weapon의 애니메이션이나 사거리, 탄수 등 총의 속성값을 들고 있습니다.
- Weapon Animation과, Ability System을 붙일 기반 작업에 들어갔습니다.



5. Ability System - 11/29 ~ 현재
- Ability System을 통해 Weapon의 격발 기능을 구현했습니다.
- 좌클릭 시 Input에 연결된 Ability가 동작하며 출발 지점과 목표 지점을 계산해서 격발하게 됩니다.


GAS(Game Ability System)를 이용한 Weapon Fire 구현

1. 파란색 Line (Camera Position 에서 Focal Position을 향하는 벡터)
2. 노란색 Line (Character Position 에서 Focal Position을 향하는 벡터)
3. 빨간색 Line (파란색 Line에 노란색 Line을 내적하여 Projection 한 것)

Weapon에서 총알이 격발되는 Ability를 연결한 후 마우스 좌클릭을 통한 Fire 테스트 진행
